Pentest Prompt Library

20 prompts para testes de penetração com etapas detalhadas

1

Reconhecimento de Domínio

Fase de Coleta de Informações:
  1. Consultar registros DNS (A, AAAA, MX, NS, TXT)
  2. Identificar subdomínios através de ferramentas passivas
  3. Coletar informações WHOIS
  4. Buscar dados em archives.org para histórico
  5. Analisar certificados SSL/TLS
2

Varredura de Portas e Serviços

Fase de Varredura:
  1. Executar varredura SYN para portas mais comuns
  2. Realizar varredura completa de todas as portas
  3. Identificar serviços e versões em portas abertas
  4. Realizar varredura de vulnerabilidades com NSE scripts
  5. Documentar descobertas para análise posterior
3

Análise de Aplicação Web

Fase de Análise:
  1. Identificar tecnologias utilizadas (Wappalyzer)
  2. Mapear estrutura e endpoints da aplicação
  3. Testar vulnerabilidades OWASP Top 10
  4. Analisar código-fonte quando disponível
  5. Testar funcionalidades de upload e entrada de dados
4

Teste de Força Bruta

Fase de Ataque:
  1. Identificar formulários de autenticação
  2. Coletar possíveis nomes de usuário
  3. Preparar listas de senhas comuns/personalizadas
  4. Configurar ferramentas de força bruta (Hydra, Burp Intruder)
  5. Executar testes controlando taxa de requisições
5

Engenharia Social - Phishing

Fase de Engenharia Social:
  1. Identitar alvos e coletar informações públicas
  2. Criar domínio similar ou confiável
  3. Desenvolver e-mail/modelo de phishing convincente
  4. Implementar mecanismo de captura de credenciais
  5. Analisar resultados e taxa de sucesso
6

Análise de Vulnerabilidades em Serviços de Rede

Fase de Análise:
  1. Identificar serviços expostos (SMB, FTP, SSH, RDP)
  2. Testar configurações padrão e autenticação nula
  3. Verificar versões vulneráveis de serviços
  4. Explorar vulnerabilidades conhecidas (metasploit)
  5. Documentar possíveis vetores de ataque
7

Teste de Vazamento de Informações

Fase de Coleta:
  1. Analisar respostas HTTP para informações sensíveis
  2. Verificar metadados em documentos públicos
  3. Testar directory listing e arquivos expostos
  4. Buscar por tokens, chaves em repositórios públicos
  5. Verificar vazamento em serviços de terceiros (Pastebin)
8

Análise de Controles de Acesso

Fase de Autorização:
  1. Mapear roles e permissões da aplicação
  2. Testar bypass de controles de acesso horizontal
  3. Testar bypass de controles de acesso vertical
  4. Verificar exposição de dados sensíveis entre contas
  5. Testar referências diretas a objetos
9

Teste de Injeção SQL

Fase de Ataque:
  1. Identificar parâmetros susceptíveis em formulários e URLs
  2. Testar com payloads básicos de detecção
  3. Determinar tipo e versão do banco de dados
  4. Explorar injeções para extrair informação
  5. Tentar elevação de privilégios via SQLi
10

Análise de Criptografia e TLS

Fase de Análise:
  1. Testar versões de protocolo suportadas
  2. Verificar cifras criptográficas implementadas
  3. Analisar certificados SSL/TLS (validade, confiança)
  4. Testar vulnerabilidades (BEAST, POODLE, etc.)
  5. Verificar implementação de HSTS
11

Teste de Cross-Site Scripting (XSS)

Fase de Ataque:
  1. Identificar pontos de entrada de dados não sanitizados
  2. Testar payloads básicos de XSS reflexivo
  3. Testar XSS armazenado em funcionalidades de persistência
  4. Verificar DOM-based XSS
  5. Explorar bypasses de filtros e WAFs
12

Análise de Configurações de Servidores

Fase de Pós-Exploração:
  1. Revisar configurações de serviços (Apache, Nginx, IIS)
  2. Verificar permissões de arquivos e diretórios
  3. Analisar logs de acesso e erro
  4. Verificar configurações de firewall internas
  5. Identificar informações de debugging expostas
13

Teste de CSRF (Cross-Site Request Forgery)

Fase de Ataque:
  1. Identificar funcionalidades críticas sem proteção CSRF
  2. Analisar tokens de anti-CSRF (geração e validação)
  3. Desenvolver PoC para explorar vulnerabilidades
  4. Testar em diferentes contextos de navegador
  5. Verificar configurações de SameSite em cookies
14

Análise de Arquivos de Configuração

Fase de Coleta:
  1. Buscar por arquivos de configuração expostos (.env, config)
  2. Analisar conteúdo em busca de credenciais
  3. Verificar permissões de acesso a arquivos sensíveis
  4. Testar directory traversal para acessar arquivos
  5. Revisar backups expostos com informações sensíveis
15

Teste de Upload de Arquivos

Fase de Ataque:
  1. Identificar funcionalidades de upload de arquivos
  2. Testar upload de tipos de arquivo perigosos
  3. Tentar bypass de verificações de tipo e conteúdo
  4. Explorar vulnerabilidades de execução de arquivos
  5. Verificar se é possível sobreescrever arquivos existentes
16

Análise de JWT Tokens

Fase de Análise:
  1. Capturar tokens JWT em tráfego de autenticação
  2. Analisar estrutura e claims do token
  3. Testar algoritmo "none" para bypass de verificação
  4. Testar weak secrets para assinatura
  5. Verificar validade e expiração de tokens
17

Teste de Business Logic Bypass

Fase de Análise:
  1. Compreender fluxos de negócio da aplicação
  2. Identificar possíveis bypasses lógicos
  3. Testar manipulação de parâmetros em processos críticos
  4. Explorar race conditions em transações
  5. Verificar validações do lado do cliente
18

Análise de Componentes de Terceiros

Fase de Análise:
  1. Identificar bibliotecas e frameworks de terceiros
  2. Verificar versões contra vulnerabilidades conhecidas
  3. Analisar componentes desatualizados ou abandonados
  4. Testar exploração de vulnerabilidades em componentes
  5. Verificar configurações padrão não alteradas
19

Teste de Session Management

Fase de Análise:
  1. Analisar cookies de sessão (segurança, escopo)
  2. Testar fixation de sessão
  3. Verificar tempo de expiração de sessões
  4. Testar logout functionality e invalidação
  5. Verificar concurrent logins e session sharing
20

Teste de APIs REST/GraphQL

Fase de Análise:
  1. Mapear endpoints e métodos disponíveis
  2. Testar autenticação e autorização em endpoints
  3. Verificar rate limiting e proteção contra abuso
  4. Testar injection em parâmetros de consulta
  5. Analizar respostas para information disclosure
